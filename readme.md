# Порівняння ефективності алгоритмів пошуку підрядка

## Опис завдання

У цьому завданні було програмно реалізовано та порівняно три алгоритми пошуку підрядка:

1. **Боєра-Мура (Boyer-Moore)**
2. **Кнута-Морріса-Пратта (KMP)**
3. **Рабіна-Карпа (Rabin-Karp)**

Для тестування використовувалися два текстових файли (`article1.txt` та `article2.txt`). Вимірювався час пошуку двох видів підрядків:

- Того, що **дійсно існує** в тексті.
- **Вигаданого**, якого немає в тексті (найгірший випадок для багатьох алгоритмів).

## Результати вимірювань часу

Нижче наведено час виконання кожного алгоритму (у секундах) згідно з отриманими результатами тестування:

| Алгоритм        | Стаття 1: Існуючий | Стаття 1: Вигаданий | Стаття 2: Існуючий | Стаття 2: Вигаданий |
| :-------------- | :----------------- | :------------------ | :----------------- | :------------------ |
| **Boyer-Moore** | 0.000007s          | 0.000022s           | 0.000012s          | 0.000044s           |
| **KMP**         | 0.000008s          | 0.000081s           | 0.000016s          | 0.000176s           |
| **Rabin-Karp**  | 0.000026s          | 0.000241s           | 0.000041s          | 0.000501s           |

## Висновки

На основі отриманих даних можна зробити наступні висновки щодо швидкодії алгоритмів.

### 1. Аналіз для Статті 1

- **Найшвидший алгоритм:** Боєра-Мура.
- При пошуку **існуючого** підрядка Боєра-Мура та KMP показали майже однакові результати (різниця всього в $1 \times 10^{-6}$ с).
- При пошуку **вигаданого** підрядка перевага Боєра-Мура стає очевидною: він майже в **4 рази швидший** за KMP і в **11 разів швидший** за Рабіна-Карпа.

### 2. Аналіз для Статті 2

- **Найшвидший алгоритм:** Боєра-Мура.
- Тенденція зберігається: алгоритм Боєра-Мура є лідером.
- Алгоритм Рабіна-Карпа показав найгірший результат, особливо на вигаданому підрядку ($0.000501$ с), що робить його найменш ефективним для цього тексту.

### 3. Загальний підсумок

1. **Алгоритм Боєра-Мура** виявився найефективнішим для обох текстів. Це пояснюється його евристикою "поганого символу", яка дозволяє алгоритму перестрибувати через частини тексту, не перевіряючи кожен символ. Чим довший підрядок і чим менше збігів, тим ефективніше він працює.
2. **Алгоритм Кнута-Морріса-Пратта (KMP)** зайняв друге місце. Він працює стабільно і швидше за наївний пошук, але в даній реалізації на Python поступається алгоритму Боєра-Мура через відсутність механізму великих "стрибків" по тексту.
3. **Алгоритм Рабіна-Карпа** виявився найповільнішим. Основна причина — накладні витрати на обчислення хеш-функції та арифметичні операції з великими числами в Python. Цей алгоритм доцільніше використовувати для специфічних задач (наприклад, пошуку множини патернів одночасно), а не для одиночного пошуку.

**Висновок:** Для задачі пошуку одного підрядка у звичайному тексті найкращим вибором є алгоритм **Боєра-Мура**.
